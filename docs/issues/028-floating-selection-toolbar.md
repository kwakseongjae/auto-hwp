# 028 — R4-1: 플로팅 셀렉션 툴바 (네이버 블로그 에디터 패턴) + "AI에게 전달"

- 상태: **open**
- 우선순위: R4-P0
- 영역: packages/react (+ editor-core 미세 — 선택 bbox getter 필요 시) + apps/hwp-lab
- 선행: 027 (done — FormatToolbar/커맨드 전부 존재). 병렬 가능: 029 (docs-only)
- 레퍼런스: 사용자 제공 네이버 블로그 에디터 스크린샷 — 선택 직상단에 붙는
  `[서체 ∨ | 크기 ∨ | B I | 색 | …]` 캡슐형 툴바, 아래꼬리(caret) 포함.

## 사용자 피드백
"드래그하면 툴바 뜨는 형식" + "AI에게 전달하기 느낌으로도 괜찮고" — 즉 이 툴바가
서식의 진입점이자 **바이브편집(선택→AI)의 진입점**이 되어야 한다.

## 목표
선택(셀/⌘다중셀/마퀴 블록)이 확정되면 **선택 영역 바로 위에 캡슐형 플로팅 툴바**가 뜬다:

```
[ 서체 ∨ | 크기 ∨ │ B  I │ 🎨글자색  🖌배경 │ ≡정렬 ∨ │ ✨ AI에게 전달 ]
                          ▼ (선택 영역을 가리키는 꼬리)
```
- 드래그/마퀴 **중에는 숨김**, 놓으면 등장. 선택 해제/Esc/문서 밖 클릭 시 사라짐.
- 위치: 선택 bbox 상단 중앙 + 8px 위. 뷰포트 상단에 잘리면 **아래로 플립**(꼬리 방향 반전),
  좌우는 뷰포트 안으로 클램프. (R13e 데스크톱에서 검증한 규칙과 동일 멘탈모델.)
- **"AI에게 전달"**: 현재 선택을 앵커 칩으로 확정하고 채팅 입력 포커스(기존 captureAnchor
  흐름 재사용 — 신규 로직 발명 금지). 이것이 이 툴바의 차별점이다.

## SDK-LAYERS 경계 (필수)
- 서식 커맨드는 **027의 core 커맨드 그대로**(formatCellRange/shadeCellRange 등) — 이 이슈는
  **표면(위치·표시·구성) 재설계**다. 새 Intent/코어 로직 금지.
- 위치 계산(선택 bbox의 페이지px→클라이언트px, 플립/클램프)은 **react 몫**(DOM) — 단 core의
  selection이 이미 mark box(페이지-로컬)를 보유하므로 그것을 소스로. 다중 선택은 마크들의
  union bbox(같은 페이지 우선, 여러 페이지에 걸치면 첫 마크의 페이지).
- FloatingToolbar는 **옵트인 컴포넌트**로 신설. 기존 FormatToolbar export는 하위호환 유지
  (deprecated 별칭 or 잔존 — README 명시). HwpWorkspace.enableEditing 경로가 새 툴바 사용.

## 구성 요소 (전부 한글 라벨/툴팁)
서체 드롭다운(FontPicker 카탈로그 연계 — 폰트명은 해당 폰트로 미리보기) · 크기 드롭다운
(9~32 프리셋+직접입력) · B · I · 글자색(스와치 팔레트+커스텀 피커) · 배경색(동일) ·
정렬(좌/중/우/양쪽) · 구분선 · **✨ AI에게 전달**. ※네이버의 인용(66)/목록/맞춤법은 우리
Intent에 대응 없음 — 스코프 밖(README에 명시, 흉내내기 금지).
문단 선택 등 미지원 조합은 027 규칙대로 **비활성+사유 툴팁**(조용한 무시 금지).

## 구현 단계
1. **위치 엔진(react 유틸)**: `computeFloatingPosition(selectionMarks, viewportRect) →
   {x, y, placement:'above'|'below'}` 순수 함수로 분리 + 단위 테스트(플립/클램프/union bbox).
2. **FloatingToolbar 컴포넌트**: 캡슐 스타일(hw-*), 꼬리, 등장/숨김 규칙(드래그 중 숨김 —
   SelectionModel의 드래그 상태 구독), 스크롤/줌 시 재배치.
3. **HwpWorkspace 배선**: enableEditing 시 기존 고정 FormatToolbar 대신 FloatingToolbar.
   "AI에게 전달" → 기존 칩 확정+채팅 포커스 경로 호출.
4. **테스트**: vitest — 위치 유틸(≥4케이스: 기본/상단플립/좌우클램프/다중마크 union) +
   툴바 표시 규칙(선택→표시, 드래그중→숨김, Esc→숨김) + AI버튼→칩 콜백. Playwright —
   기존 editing 4시나리오를 새 툴바 경유로 갱신·통과 + 신규 1: "드래그 선택→툴바가 선택
   근처에 표시(bbox 거리 assert)→B 적용→AI에게 전달→칩 확정".
5. 게이트 1회(엔진 무접촉 확인용) + 기존 e2e 전부.

## 함정
- 줌 상태에서 페이지px→클라이언트px 변환이 어긋나기 쉽다 — HwpPageView의 기존 좌표 변환
  단일 지점을 재사용하라(새 변환 산술 금지).
- 툴바 위 클릭이 선택 해제(바깥 클릭 판정)로 새면 안 된다 — 툴바는 "안쪽" 영역.
- 드롭다운 열림 중 선택이 바뀌면 드롭다운 닫고 재배치.
- 색 피커는 onChange만(드래그 스팸 금지 — R13d 교훈).
