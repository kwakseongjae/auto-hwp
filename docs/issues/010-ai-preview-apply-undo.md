# 010 — P1-B: AI 편집 프리뷰→적용 게이트 + undo

- 상태: **open**
- 우선순위: P1 (목표 1의 킬러 루프 2/3 — 신뢰가 곧 사용성)
- 영역: 데스크톱 UX / 바이브편집
- 선행: 009 (앵커 칩)
- 레드팀: **R5** (인젝션의 실질 방어선 = 사람의 승인 게이트)

## 목표
지금 AI 편집은 즉시 커밋된다. 이를 **"제안 → 문서 위 하이라이트 프리뷰 → ✓ 적용 / ✗ 취소"**
2단계로 바꾼다. 부수적으로 편집 히스토리 undo(⌘Z)를 사용자에게 노출한다.
악성/오작동 편집이 문서를 조용히 바꾸는 것을 막는 유일한 구조적 방어선이며(R5),
동시에 "AI가 뭘 했는지 보인다"는 신뢰 UX의 핵심이다.

## 컨텍스트
- `hwp-mcp`에 이미 `undo`/`redo` 툴이 있다(세션이 `apply_content`로 변이되고 되돌릴 수
  있는 구조). Tauri 쪽에서 이 undo가 UI에 노출돼 있는지 먼저 확인하라 — **조사 단계에서
  실제 코드를 읽고, 없으면 만들고 있으면 재사용**한다.
- `PendingInline.tsx`가 존재한다 — 이름상 "펜딩 편집" UI의 흔적. 먼저 읽고 재사용
  가능성을 판단하라(문서와 코드가 어긋나면 멈추고 보고 — 공통 계약 §4.3).
- 자체 렌더는 diff refresh(no-op skip)를 지원(R9 완료) — "적용 전/후" 재렌더 비용은 낮다.

## 파일 지도
- `crates/hwp-viewer/src/lib.rs` — ai_edit 커맨드, undo/redo 배선 지점
- `crates/hwp-mcp/src/lib.rs` — Session의 undo 스택
- `crates/hwp-viewer/ui/src/Chat.tsx` — 제안 카드(적용/취소 버튼)
- `crates/hwp-viewer/ui/src/App.tsx` — 프리뷰 하이라이트 오버레이, ⌘Z 키맵
- `crates/hwp-viewer/ui/src/PendingInline.tsx` — 기존 펜딩 UI (재사용 판단)

## 구현 단계
1. **제안 모드**: `ai_edit`에 `dry_run` 플래그 추가 — LLM이 만든 EditScript를 적용하지
   않고 (a) 사람이 읽을 요약(어느 블록, 무엇을), (b) 영향 블록 좌표 목록을 반환.
   구현 전략은 둘 중 하나를 조사 후 선택하고 근거를 보고에 남겨라:
   - (i) 진짜 dry-run: script를 들고만 있다가 승인 시 적용 (상태 관리 필요)
   - (ii) 적용+표시 후 거부 시 undo: 기존 undo 스택 재사용 (구현 최소, 단 "적용 전
     프리뷰"가 아니라 "적용 후 확인"이 됨 — 렌더로 보여줄 수 있는 장점)
   권장은 (ii)로 시작(부품 재사용, 실패 모드 단순), 카드 문구로 "적용됨 — 되돌리기"
   대신 "제안 반영됨 ✓ 유지 / ↩ 되돌리기"를 명확히.
2. **하이라이트**: 영향 블록들을 자체 렌더 위 오버레이로 하이라이트(기존 선택 오버레이
   스타일 재사용, 색만 구분). 채팅 카드 hover ↔ 문서 하이라이트 연동.
3. **undo 노출**: ⌘Z/⇧⌘Z를 문서 포커스에서 세션 undo/redo로 배선. WYSIWYG 에디터가
   열려 있는 동안은 contentEditable 자체 undo와 충돌하므로 **에디터 열림 중엔 세션
   undo를 건드리지 마라**(에디터 닫힘 상태에서만 동작).
4. **undo 스택 상한**: R13 — 스택 N=50 상한을 세션에 넣어라(무한 성장 방지).

## 검증
- 공통 스위트(§4.2) + UI 빌드.
- 수동: ① 앵커 칩+프롬프트 → 제안 카드 + 문서 하이라이트 → ✓ 유지 ② 같은 흐름 → ↩ →
  문서가 바이트 단위로 원상복구(export-html 출력 비교로 확인) ③ 연속 편집 3회 후 ⌘Z 3회 →
  전부 역순 복구 ④ 에디터 열림 중 ⌘Z가 세션 undo를 오발하지 않음.

## 수용 기준
- [ ] AI 편집이 제안 카드+하이라이트를 거쳐 유지/되돌리기 가능
- [ ] 되돌리기 후 문서 상태가 편집 전과 동일 (export 비교로 검증)
- [ ] ⌘Z/⇧⌘Z 세션 undo/redo (에디터 열림 중 제외)
- [ ] undo 스택 상한 50
- [ ] 게이트/타입체크/빌드 그린

## 함정
- undo가 "텍스트만" 되돌리고 서식/음영 op를 못 되돌리는 부분 구현이면 반쪽이다 —
  Session 변이가 스냅샷 기반인지 op 역연산 기반인지 조사 단계에서 확정하고 보고하라.
- 하이라이트 오버레이는 px 좌표(리페인트마다 재계산 필요) — 페이지 분할 표의 조각은
  (section,block)이 같아도 페이지가 다르다. 블록→페이지 매핑은 기존 `block_pages` 재사용.
- 프리뷰 상태에서 사용자가 수동 편집(WYSIWYG)을 시작하면 제안 상태를 정리(취소)하고
  토스트로 알릴 것 — 두 변이 경로가 겹치면 안 된다.
