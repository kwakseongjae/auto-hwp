# 049 — R11-5: 이미지 이동/리사이즈 SDK 승격 — 선택 핸들 + 드래그

- 상태: **open** · 우선순위: R11-P1 · 영역: packages/react(신규 ImageOverlay + HwpWorkspace 이미지 선택/오버레이 영역) + editor-core(이미지 커맨드) + 필요 시 wasm additive
- 병렬: 050(이미지 삽입 — 드롭 존/업로드 버튼/삽입 intent 영역 소유). 삽입 경로 접촉 금지.

## 근거 (감사표 "이미지 이동/리사이즈" M — 데스크톱 M1/M4)
데스크톱: image_at/image_bbox 질의 + SetImageSize/move 계열 op로 클릭 선택→핸들→이동/리사이즈.
SDK: hitTest가 이미지 블록을 인지하나(038 호버가 kind 표시) 선택 후 조작 UI가 없다.

## 목표
- 이미지 클릭 = 이미지 선택(모서리 4+변 4 핸들 오버레이, 피그마식) → **드래그 = 이동,
  핸들 드래그 = 리사이즈**(모서리는 종횡비 유지, Shift로 해제 — 피그마 관례).
- 커밋은 기존 op 경로(이동/크기 intent — wasm 동등물 실측, 없으면 additive 바인딩+게이트).
  적용-확인(031): 커밋 후 image_bbox 재조회로 실제 변화 검증, 무변화 = 정직한 실패.
- 라이브 프리뷰: 드래그 중 오버레이 고스트만 이동(030 rAF+ref, 렌더 0) — SVG 재렌더는 커밋 후 1회.
- undo 1단위, 038 호버·039 메뉴와 공존(이미지 위 메뉴에 크기 초기화 항목은 재량).

## 실측 출발점
- 데스크톱: crates/hwp-viewer image_at/image_bbox/set_image_size/move_image 커맨드 + ui 핸들 UI(M4).
- ⚠️ 좌표: own-render 지오메트리는 px, 커밋 ops는 HWPUNIT(§4.5) — image_bbox/커밋의 단위를
  실측하고 변환은 units.ts 한 곳.
- 분할표 안 이미지·페이지 경계 이미지 동작 실측(리사이즈로 페이지가 밀리는 리플로우 케이스 —
  LOCKSTEP 게이트 불변 확인 필수).
