# 057 — R12 버그: HWPX 익스포터 표 앵커링 (셀 편집 후 표가 문서 끝으로 오배치)

- 상태: open · 우선순위: R12-P1(정확성 버그) · 영역: crates(HWPX 재방출 — hwp-hwpx serialize/dirty 재방출 계열)
- 발견 경위: 052 golden 복구 테스트가 실측으로 격리(2026-07-10). 재현 절차:
  `apps/hwp-lab/src/lib/goldenRecovery.test.ts` 상단 매트릭스.

## 증상 (실측)
- **hwpx 오리진** 문서에서 `SetTableCellRuns`(표 셀 편집) 후 `toHwpx()` 재방출 → 재열기하면
  **편집된 표가 문서 끝으로 오배치**되고, 원 위치에는 편집 전 표가 그대로 남는다.
- `.hwp` 오리진에서는 제자리 보존(비발현). 문단 편집(Replace) 레인은 hwpx 오리진에서도 정상
  (052 golden이 전 페이지 pageSvg 동일로 잠금).
- 사용자 영향: **편집 후 HWPX 저장 일반 경로**의 문서 구조 파손 + 자동저장 복구본을 다시
  표 편집→재복구하는 2세대 시나리오에서 발현. dirty-only 재방출의 앵커링 결함으로 추정.

## 목표
hwpx 오리진 + 표 셀 편집 + 재방출에서 표가 원 위치에 유지된다. 052 golden 매트릭스의 해당
케이스를 "콘텐츠 보존" 잠금에서 "전 페이지 pageSvg 동일"로 승격.

## 설계 (조사 먼저)
1. goldenRecovery 매트릭스 절차로 Rust 단 재현 테스트를 먼저 작성(hwp-hwpx 또는 hwp-mcp).
2. 원인 후보: dirty 표 블록을 verbatim 스트림의 원 위치에서 치환하지 않고 끝에 append하는
   재방출 순서/인덱스 결함 — dirty 마킹→재방출 경로의 verbatim/재합성 분기 추적.
3. 수정 후: 라운드트립 골든(전 페이지 SVG 동일) + 기존 바이트 안정 골든 무회귀.
