## 목표
우클릭 → 그 지점을 hit-test → 대상별 메뉴:
- **셀**: 텍스트 편집(032 제자리 에디터), 굵게/배경색(028과 동일 액션), 행 삽입/삭제·열 삽입/삭제
  (기존 intent 있는 것만 — 없는 액션은 넣지 마라, 실측 후 구성), ✨AI에게 전달(028 경로)
- **문단**: 텍스트 편집, ✨AI에게 전달
- **바탕(비개체)**: 표 추가(027 픽커), 붙여넣기류 금지(클립보드 스코프 아웃)
메뉴는 뷰포트 경계 클램프, Esc/외부클릭/스크롤로 닫힘, 키보드 ↑↓/Enter 선택.

## 설계
- 액션 로직 신설 금지: 전부 **기존 경로 위임** — core.edit(EditController)·028 FloatingToolbar가
  쓰는 동일 핸들러를 공용 유틸로 추출해 양쪽이 공유(중복 코드 금지, 추출 시 028 무회귀).
- 우클릭 지점 → 선택 갱신(그 자리를 클릭한 것과 동일한 선택 규칙 023/021) 후 메뉴 오픈 —
  메뉴 액션은 항상 "현재 선택" 대상으로 동작(028과 같은 계약).
- 038의 HoverLayer/커서와 공존: 메뉴 열림 중 호버 억제.

## 구현 단계
1. HwpWorkspace 기존 액션 핸들러 인벤토리(028 툴바·상단 툴바) → 공용 액션 유틸 추출(순수 위임).

   (preventDefault, 시트 위에서만).
3. 테스트: vitest(메뉴 구성 대상별 분기·클램프·닫힘 규칙·액션 위임 mock), e2e 1개(셀 우클릭 →
