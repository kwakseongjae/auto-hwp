# 033 — R5-4: 피그마급 UX 격차 리서치 → 실행 가능한 백로그

- 상태: **open** · 우선순위: R5-P1 · 영역: **docs-only** (산출물: docs/FIGMA-GRADE-UX.md)
- 병렬: 030·031과 완전 disjoint

## 배경 (사용자 북극성)
"성능적으로나 경험적으로나 — 수동 선택·편집 포함 — **피그마 수준**." 파싱 최적화·렌더링
최적화·인터랙션 전반에서 격차를 체계적으로 조사해, **이후 workflow가 바로 집을 수 있는
빌더블 백로그**로 정리한다. 코드 수정 없음 — 그러나 모든 주장에 **현 코드 실측 근거**를 달 것.

## 산출물: docs/FIGMA-GRADE-UX.md 구조
1. **레이턴시 예산표**: 상호작용별 목표(클릭→선택 <16ms, 드래그 프레임 <16ms, 편집 진입
   <100ms, 문서 열기 <1s/10p, 편집 적용→화면 <100ms)와 **현재 실측치**(025 벤치·030 결과
   인용 + 부족분은 직접 측정 스크립트로) — 격차가 있는 항목만 백로그화.
2. **렌더링 아키텍처 격차**: 현재(페이지=거대 인라인 SVG DOM) vs 피그마(캔버스/GPU).
   조사 항목: (a) SVG DOM 한계 실측 — 페이지당 노드 수, 스크롤/줌 프레임, 어디까지 버티나
   (benchmark2 25p로) (b) 부분 갱신 — 편집 페이지만 재주입(현재 전 페이지 문자열 재조회?
   실측) (c) 페이지 가상화(뷰포트 밖 언마운트) (d) 장기 옵션: 캔버스 렌더러(엔진이 PaintOp
   IR을 이미 갖고 있음 — hwp-render에 캔버스 싱크 추가 타당성, wasm에서 직접 캔버스 페인트
   경로) — 각각 난이도/효과/리스크.
3. **인터랙션 격차 카탈로그**: 피그마 기준으로 우리에게 없는 것 — 스페이스+드래그 팬,
   핀치/⌘휠 줌(커서 중심), 더블클릭 심도 진입(표→셀→텍스트), 방향키 셀 이동, 다중 선택
   후 일괄 이동, 호버 프리하이라이트, 컨텍스트 메뉴, 스마트 가이드/스냅, 커서 상태 체계
   (default/text/col-resize/row-resize/grab) — 각 항목: 우리 엔진 부품으로 가능한가(기존
   Intent/지오메트리 매핑), 규모(S/M/L).
4. **파싱/열기 격차**: 대형 문서 열기 시간 실측(benchmark2, 가능하면 더 큰 실물), wasm
   스트리밍/워커화(메인스레드 블로킹 실측 — open이 UI를 얼마나 멈추나), 폰트 로딩 병렬화.
5. **텍스트 편집 심화 로드맵**: 제자리 편집(032) 다음 단계 — 부분 서식(런 단위) 편집,
   캐럿/선택 렌더(엔진 hit-test 글리프 단위 — 데스크톱 caret 자산 재사용 가능성), IME
   인라인 조합 표시. 데스크톱 richedit의 교훈(gotcha 목록) 링크.
6. **백로그 표**: 위 전부를 [ID | 제목 | 근거(실측) | 효과 | 난이도 | 의존 | 빌더블 이슈로
   승격 시 스코프 한 줄]로 — 우선순위 제안 포함. **각 행은 034+ 이슈로 바로 복사 가능한
   수준**이어야 한다.

## 규율
- "될 것이다" 금지 — 실측(간단 스크립트/Playwright 트레이스/노드 카운트) 또는 코드 인용.
  측정 스크립트는 scripts/ 또는 bench/에 tracked로 남겨라(코드 무수정, 추가만).
- 피그마 동작 서술은 일반 지식으로 충분(웹 조사 불요) — 우리 쪽 격차 실측이 본체다.
- 030·031이 병렬로 고치는 항목은 "진행 중(030/031)"으로 표기하고 중복 조사 금지.
